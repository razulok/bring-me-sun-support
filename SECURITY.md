# Sicherheitsrichtlinie / Security Policy

## Deutsch

### Öffentliche Fehlerberichte und Vorschläge

Gewöhnliche, nicht vertrauliche Fehlerberichte und Funktionsvorschläge gehören in die [öffentlichen GitHub-Issues](https://github.com/razulok/bring-me-sun-support/issues/new/choose).

Veröffentliche dort niemals:

- personenbezogene Daten oder genaue Privatadressen,
- Passwörter, Zugangsdaten, API-Schlüssel, Zugriffstoken oder andere Geheimnisse,
- Schwachstellen-, Exploit- oder andere Details, die ein Sicherheits- oder Datenschutzrisiko offenlegen könnten.

### Vertrauliche Sicherheits- und Datenschutzmeldungen

Öffne für vermutete Sicherheitslücken, Datenschutzprobleme oder andere vertrauliche Meldungen kein öffentliches Issue. Sende sie stattdessen über [GitHub Private Vulnerability Reporting](https://github.com/razulok/bring-me-sun-support/security/advisories/new). Teile dort nur die Informationen, die zur vertraulichen Untersuchung erforderlich sind.

## English

### Public bug reports and feature requests

Ordinary, non-sensitive bug reports and feature requests belong in the [public GitHub issues](https://github.com/razulok/bring-me-sun-support/issues/new/choose).

Never publish any of the following there:

- personal data or precise home addresses,
- passwords, credentials, API keys, access tokens, or other secrets,
- vulnerability, exploit, or other details that could disclose a security or privacy risk.

### Confidential security and privacy reports

Do not open a public issue for suspected vulnerabilities, privacy concerns, or other confidential reports. Submit them through [GitHub Private Vulnerability Reporting](https://github.com/razulok/bring-me-sun-support/security/advisories/new) instead. Share only the information required for confidential investigation.
